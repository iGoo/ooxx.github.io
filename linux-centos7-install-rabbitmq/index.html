出现如下错误

从错误可以看到重要的一段是/usr/bin/ld: cannot find -lrabbitmq。意思就是无法找到我们上面安装的rabbitmq-c库。然后我去/usr/local/rabbitmq-c目录里执行ls命令后发现也确实只有include和lib目录。

到这里我真的是百思不得其解，然后我又试着查看了lib目录里的内容，发现竟然还有个目录，目录名为x86_64-linux-gnu在这里目录里才是我们要的库文件

于是我执行了cp -r x86_64-linux-gnu/* .把这里目录下的所有文件拷贝到上一层。然后继续执行

发现可以正确的安装了。

小结：
之前是在CentOS上安装的，最近使用Homestead开发环境，用的是Ubuntu16.04。系统不一样导致编译出的rabbitma-c目录下多了平台相关的文件夹。而CentOS上则是直接编译到了rabbitmq-c根目录下。
